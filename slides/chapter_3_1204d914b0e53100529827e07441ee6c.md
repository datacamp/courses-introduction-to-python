```py

```{{3}}

```out
Help on built-in function round in module builtins:

round(number, ndigits=None)
    Round a number to a given precision in decimal digits.
    
    The return value is an integer if ndigits is omitted or None. 
    Otherwise the return value has the same type as the number. ndigits may be negative. 
```{{3}}

`@script`
Nog zo’n ingebouwde functie is round. Die neemt twee inputs: eerst een getal dat je wilt afronden, en ten tweede de precisie waarmee je afrondt, dus hoeveel cijfers na de komma je wilt behouden. Stel dat je één-komma-achtenzestig wilt afronden op één decimaal. De eerste input is één-komma-achtenzestig, de tweede input is één. Je scheidt de inputs met een komma.

Maar er is meer. Het is mogelijk om de functie round met maar één input aan te roepen, zo. Dit keer merkt Python dat je de tweede input niet hebt meegegeven, en kiest automatisch om het getal af te ronden naar het dichtstbijzijnde gehele getal.

Om te begrijpen waarom beide aanpakken werken, openen we de documentatie. Dat kan met nog een functie, help.

Het blijkt dat round twee inputs neemt.
