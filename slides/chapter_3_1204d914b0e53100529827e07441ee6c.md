Help on built-in function round in module builtins:

round(number, ndigits=None)
    Round a number to a given precision in decimal digits.
    
    The return value is an integer if ndigits is omitted or None. 
    Otherwise the return value has the same type as the number. ndigits may be negative. 
```{{3}}

`@script`
Outra função integrada é round. Ela recebe duas entradas: primeiro, o número que você quer arredondar e, depois, a precisão do arredondamento, ou seja, quantos dígitos depois do separador decimal você quer manter. Suponha que você queira arredondar um ponto sessenta e oito com uma casa decimal. A primeira entrada é um ponto sessenta e oito. A segunda é um. As entradas são separadas por vírgula.

Mas tem mais. É perfeitamente possível chamar a função round com apenas uma entrada, deste jeito. Desta vez, o Python percebeu que você não especificou a segunda entrada e automaticamente optou por arredondar o número para o inteiro mais próximo.

Para entender por que as duas abordagens funcionam, vamos abrir a documentação. Podemos fazer isso com outra função, help, deste jeito.

Parece que round recebe duas entradas.
