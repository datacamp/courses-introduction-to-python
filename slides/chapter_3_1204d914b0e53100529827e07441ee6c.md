`@script`
Funktionen sind für dich eigentlich nichts Neues: Du hast sie bereits benutzt. „type“ ist zum Beispiel eine Funktion, die den Typ eines Werts zurückgibt. Aber was ist eine Funktion? Einfach gesagt ist eine Funktion ein Stück wiederverwendbarer Code, der eine bestimmte Aufgabe erledigen soll. Du kannst Funktionen aufrufen, anstatt selbst Code dafür schreiben zu müssen. Vielleicht kann ein Beispiel das Ganze besser erklären.

```py

```{{3}}

```out
Help on built-in function round in module builtins:

round(number, ndigits=None)
    Round a number to a given precision in decimal digits.
    
    The return value is an integer if ndigits is omitted or None. 
    Otherwise the return value has the same type as the number. ndigits may be negative. 
```{{3}}

`@script`
Eine weitere integrierte Funktion ist „round“. Es braucht zwei Eingaben: erstens die Zahl, die du runden willst, und zweitens die Genauigkeit, mit der gerundet werden soll: Also wie viele Stellen nach dem Komma du nach der Rundung behalten willst. Angenommen, du willst „eins-Komma-sechs-acht“ auf eine Dezimalstelle runden. Der erste Eingabewert ist „eins-Komma-sechs-acht“, der zweite ist „eins“. Die Eingaben trennt man mit einem Komma.

Aber das war nicht alles. Es ist möglich, die Funktion „round“ mit nur einer Eingabe aufzurufen, so wie hier. Dieses mal hat Python erkannt, dass du die zweite Eingabe nicht angegeben hast, und rundet die Zahl automatisch auf die nächste ganze Zahl.

Um zu verstehen, warum beide Ansätze funktionieren, schauen wir uns die Dokumentation an. Das kannst du mit der Funktion namens „help“ tun.

Es sieht so aus, als ob „round“ zwei Eingaben benötigt.
