`@script`
Python gebruikt de index hiervoor. Kijk hier nog eens naar de lijst fam. Het eerste element in de lijst heeft index nul, het tweede element heeft index één, enzovoort. Stel dat je de lengte van emma wilt selecteren, de float één-komma-achtenzestig. Dat is het vierde element, dus het heeft index drie. Om het te selecteren, gebruik je drie tussen vierkante haken.

Op dezelfde manier selecteer je de string "dad" uit de lijst,

![De syntaxis voor het opsplitsen van Python-lijsten, waarbij je ziet dat de startwaarde wel meetelt in de subset, maar de eindwaarde niet. (https://assets.datacamp.com/production/repositories/288/datasets/83dd2f807be0d4d08a187935eed11667c18fcfe3/slicing-syntax.png = 40){{3}}

`@script`
waarmee je meerdere elementen uit een lijst kunt selecteren en zo een nieuwe lijst maakt. Dat doe je door een bereik op te geven met een dubbele punt. Laten we eerst nog een keer naar de lijst kijken en dan dit stukje code proberen.

Kun je raden wat dit oplevert? Misschien een lijst met de float één-komma-achtenzestig, de string "mom" en de float één-komma-een-en-zeventig, overeenkomend met het vierde, vijfde en zesde element in de lijst? Laten we de output bekijken.

Blijkbaar worden alleen de elementen met index drie en vier teruggegeven. Het element met index vijf wordt niet meegenomen. In het algemeen is dit de syntaxis: de index die je vóór de dubbele punt opgeeft, dus waar de slice start, wordt inbegrepen, terwijl de index na de dubbele punt, waar de slice eindigt, niet wordt inbegrepen.

Met dit in gedachten: kun je zeggen wat deze aanroep zal teruggeven?

Waarschijnlijk heb je goed geraden dat deze aanroep je een lijst met drie elementen geeft, overeenkomend met de elementen met index één, twee en drie van de lijst fam.

Je kunt er ook voor kiezen om de index vóór of na de dubbele punt weg te laten.
