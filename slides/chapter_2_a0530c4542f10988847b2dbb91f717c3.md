```{{2}}

- Dá nome a um conjunto de valores{{3}}

- Contém qualquer tipo{{4}}

- Contém diferentes tipos{{5}}

`@script`
Você pode criar essa lista usando colchetes. Imagine que você perguntou para suas duas irmãs e seus pais qual é a altura deles, em metros. Você pode criar a lista assim.

É claro que também dá para usar uma variável para fazer referência a essa estrutura de dados. Basta colocar o nome da variável e o sinal de igual na frente, desta forma.

Uma lista é uma maneira de dar um único nome a um conjunto de valores. Esses valores, ou elementos, podem ser de qualquer tipo; podem ser floats, inteiros, booleanos, strings, mas também tipos Python mais avançados, até mesmo listas.

É perfeitamente possível que uma lista também tenha tipos diferentes.

```yaml
type: FullSlide
key: 35d6825cd6

`@part1`
- `[a, b, c]`

```py
fam = ["liz", 1.73, "emma", 1.68, "mom", 1.71, "dad", 1.89]
```
```py
fam
```

```out
['liz', 1.73, 'emma', 1.68, 'mom', 1.71, 'dad', 1.89]
```

```py
fam2 = [["liz", 1.73],
		["emma", 1.68],
		["mom", 1.71],
		["dad", 1.89]]
```{{1}}
```py
fam2
```{{2}}

```out
[['liz', 1.73], ['emma', 1.68], ['mom', 1.71], ['dad', 1.89]]
```{{2}}

`@script`
Suponha que você queira colocar o nome das suas irmãs e pais na lista, para saber de quem é cada altura. Você pode incluir algumas strings sem problemas.

Mas isso não é tudo. Acabei de dizer que as listas também podem ter outras listas dentro delas. Em vez de colocar as strings entre os números, você pode criar pequenas sublistas para cada pessoa da família. Uma para Liz, uma para Emma e assim por diante. Agora, você pode dizer ao Python que essas sublistas são os elementos de outra lista, que chamei de fam2: as pequenas listas são colocadas entre colchetes e separadas por vírgula. Agora, se você imprimir fam2, vai ver que temos uma lista de listas. A lista principal contém quatro sublistas.

Aqui estamos lidando com um novo tipo Python, além das strings, booleanos, inteiros e floats que você já conhece:
