`@script`
Supongamos que quieres obtener el índice de la cadena mom en la lista fam. fam es un objeto de Python con el tipo lista y tiene un método llamado index. Para llamar al método, utiliza la notación de punto, como se muestra a continuación. La única entrada es la cadena mom, el elemento para el que deseas obtener el índice.

Python devuelve 4, que es efectivamente el índice de la cadena mom. He llamado al método index on en la lista fam, y el resultado ha sido 4. Del mismo modo, puedo utilizar el método count en la lista fam para contar el número de veces que aparece 1,73 en la lista.

Python me da 1, lo cual tiene sentido, ya que solo Liz mide 1,73 metros.

`@script`
hay una cosa más que quiero decirte. Algunos métodos pueden cambiar los objetos sobre los que se llaman. Recuperemos la lista fam y llamemos al método append que incluye. Como entrada, pasamos una cadena que queremos añadir a la lista.

Python no genera una salida, pero, si volvemos a comprobar la lista fam, vemos que se ha ampliado con la cadena me.

Hagámoslo de nuevo, esta vez para añadir mi altura a la lista.

Una vez más, la lista fam se ha ampliado.

Esto es muy interesante, ya que es posible escribir código muy conciso para actualizar las estructuras de datos sobre la marcha, pero también puede ser bastante peligroso. Algunas llamadas a métodos no modifican el objeto al que se llaman, mientras que otras sí lo hacen; ten cuidado.
