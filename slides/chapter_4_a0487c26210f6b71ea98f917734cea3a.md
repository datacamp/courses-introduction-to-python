今度はうまくいきました。計算は要素ごとに実行されています。1人目の BMI は、np_weight の1つ目の要素を np_height の1つ目の要素の2乗で割って計算され、2人目は2つ目の要素同士で、というように続きます。

## NumPyの注意点

```yaml

```

`@part1`
```py
np.array([1.0, "is", True])
```{{1}}

```out
array(['1.0', 'is', 'True'], dtype='<U32')
```{{1}}

- NumPy配列: 型は1種類のみ{{1}}

`@script`
ただし、いくつか注意点があります。まず、NumPy がここまで簡単に処理できるのは、NumPy 配列が単一の型の値だけを含むと仮定しているからです。浮動小数点数の配列か、ブール値の配列か、という具合です。もしこのように異なる型を混ぜて配列を作ろうとすると、結果の NumPy 配列は1つの型に統一されます。この場合は文字列になり、ブール値と浮動小数点数はどちらも文字列に変換されます。

次に、NumPy 配列は、これまで出てきた浮動小数点数、文字列、リストと同様に、Python の新しい型のひとつにすぎないという点も知っておきましょう。つまり、独自のメソッドを持っており、期待と異なる挙動をすることがあります。

異なる Python の型を扱うときは、結果が大きく変わることがあるので、しっかり注意してください。

## NumPyのサブセット化

```yaml
type: FullSlide
key: c1f3774f83
code_zoom: 71
```

`@part1`
```py
bmi
```{{1}}

```out
array([21.85171573, 20.97505669, 21.75028214, 24.7473475 , 21.44127836])
```{{1}}

```py
bmi[1]
```{{2}}

```out
20.975
```{{2}}

```py
bmi > 23
```{{3}}

```out
array([False, False, False,  True, False])
```{{3}}

```py
bmi[bmi > 23]
```{{4}}

```out
array([24.7473475])
```{{4}}

`@script`
NumPy 配列は、基本的には通常の Python リストとほぼ同じように扱えます。たとえば配列から要素を取り出すときは、角かっこを使います。2人目、つまりインデックス 1 の BMI を取り出したい場合は、これで大丈夫です。

NumPy には、もう1つの抽出方法として、ブール値の配列を使うやり方もあります。bmi 配列のうち 23 を超える値だけを取り出したいとしましょう。まずはこのように、不等号を使います。

結果はブール値の NumPy 配列になります。対応する bmi が 23 より大きければ True、小さければ False です。次に、このブール配列を角かっこの中に使って抽出します。bmi の中で 23 を超える要素、つまり対応するブール値が True のものだけが選ばれます。23 を超える BMI は1つだけなので、その特定の BMI を1要素だけ含む NumPy 配列が得られます。

比較結果を使ってデータを選び出すのは、驚くような発見につながる、とても一般的な手法です。
